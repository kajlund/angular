The Controller As syntax

Since AngularJS 1.2+ it has been recommended that instead of attaching properties to the $scope object, we should instead use the ControllerAs syntax.

Think of the declaration "MainController as main" as a declaration like var main = new MainController() creating an alias for the controller that you use in your view.

Another plus of using this syntax is that you can separate parent/child controllers in a view by name. Also, you won't end up with a situation whereby a child controller property overwrites a parent controller property with the same name.
